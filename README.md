# CallCRM — установка сервера (подробно, для всех)

Это серверная часть: сайт-CRM, куда приходят записи разговоров, где они
превращаются в текст, делятся по спикерам и анализируются (ищутся инциденты —
угрозы, конфликты и т.п.).

> ⚠️ Эта инструкция написана максимально пошагово. Просто делай по порядку.
> Команды (то, что в серых рамках) нужно **скопировать и вставить в Терминал**,
> затем нажать **Enter**.

---

## Что нужно один раз поставить

### Шаг 1. Установить Docker Desktop
Docker — это программа, которая запускает CallCRM «в коробке», чтобы ничего
вручную не настраивать.

1. Открой сайт: **https://www.docker.com/products/docker-desktop/**
2. Нажми синюю кнопку **Download Docker Desktop** и выбери свою систему:
   - **Mac с процессором Apple (M1/M2/M3/M4)** → «Download for Mac - Apple Chip»
   - **Mac на Intel** → «Download for Mac - Intel Chip»
   - **Windows** → «Download for Windows»
   - *(Не знаешь, какой у тебя Mac? Меню Apple слева вверху → «Об этом Mac» →
     если написано Apple M… — это Apple Chip.)*
3. Открой скачанный файл и установи (на Mac — перетащи Docker в папку
   «Программы»; на Windows — жми «Далее/Next» до конца).
4. **Запусти Docker Desktop** (иконка кита 🐳). Подожди, пока кит вверху экрана
   перестанет «мигать» и станет статичным — значит Docker готов.

> Docker должен быть **запущен всегда**, когда работает CallCRM. Если выключишь
> Docker — CRM перестанет работать.

### Шаг 2. Получить папку с проектом
У тебя уже должна быть папка **CallCRM** на рабочем столе. Если её нет —
попроси её у того, кто давал проект (или распакуй присланный архив на рабочий
стол).

### Шаг 3. Открыть Терминал в этой папке
- **Mac:** нажми `Cmd + Пробел`, напечатай `Терминал`, нажми Enter. Откроется
  чёрное/белое окно. Вставь туда команду и нажми Enter:
  ```bash
  cd ~/Desktop/CallCRM
  ```
- **Windows:** открой папку CallCRM в Проводнике, в адресной строке сверху
  напечатай `powershell` и нажми Enter — откроется синее окно уже в нужной папке.

> `cd` означает «перейти в папку». После этой команды Терминал «находится»
> внутри папки CallCRM, и все следующие команды будут выполняться там.

## Запуск (первый раз)

### Шаг 4. Запустить все сервисы
Вставь в Терминал и нажми Enter:
```bash
docker compose up -d --build
```
Первый раз это **долго** (5–15 минут) — Docker скачивает и собирает всё нужное.
Это нормально. Жди, пока снова появится строка для ввода.

Проверить, что всё поднялось:
```bash
docker compose ps
```
Должен увидеть список из 7 строк (backend, celery, frontend, ingest, ollama,
postgres, redis) со статусом **Up**.

### Шаг 5. Скачать «мозги» (модели распознавания и анализа)
Это нужно один раз. Скачается ~4 ГБ — займёт время, зависит от интернета.

Скачать модель распознавания речи:
```bash
docker compose run --rm backend python manage.py download_models
```
Скачать модель анализа (поиск инцидентов):
```bash
docker compose exec ollama ollama pull qwen2.5:1.5b
```
Жди надписи **success / Модели готовы** после каждой команды.

### Шаг 6. Создать вход администратора
```bash
docker compose exec backend python manage.py createsuperuser
```
Терминал спросит по очереди:
- **Username** — придумай логин (например `admin`), Enter.
- **Email** — можно оставить пустым, Enter.
- **Password** — придумай пароль (при вводе он **не показывается** — это норма),
  Enter, и **повтори** его.
- Если спросит «Bypass password validation?» — напечатай `y`, Enter.

Запиши логин и пароль — они нужны для входа на сайт.

## Как зайти в CRM

1. Открой браузер (Chrome / Safari / Edge).
2. В адресной строке набери: **http://localhost:3000** и нажми Enter.
3. Введи логин и пароль из Шага 6.

Готово — ты внутри CRM.

## Что сделать внутри CRM перед подключением записывающего приложения

Записывающее приложение (на компьютере сотрудника) привязывается к компании по
**ключу компании**. Получи его так:

1. Слева в меню нажми **«Управление»**.
2. Вкладка **«Компании»** → создай компанию (или открой существующую).
3. Нажми **«Ключи»** — там длинная строка **API Key**. Это и есть **ключ
   компании**. Скопируй её (она понадобится в записывающем приложении).
4. Вкладка **«Сотрудники»** → создай сотрудника: впиши ФИО, **email** и
   **пароль**. Этот email и пароль сотрудник введёт в записывающем приложении.

> Итог: для записывающего приложения нужны три вещи —
> **адрес сервера**, **ключ компании**, **email и пароль сотрудника**.

## Каждый день: включить и выключить

**Включить** (Docker Desktop должен быть запущен):
```bash
cd ~/Desktop/CallCRM
docker compose up -d
```
Через ~30–60 секунд открывай http://localhost:3000.

**Выключить:**
```bash
cd ~/Desktop/CallCRM
docker compose stop
```

## Если что-то не работает

| Проблема | Что делать |
|----------|-----------|
| Браузер пишет «Не удаётся подключиться» к localhost:3000 | Проверь, что Docker Desktop запущен (кит 🐳 вверху). Подожди минуту после `up -d`. |
| Команда `docker` пишет «command not found» / «не является командой» | Docker Desktop не установлен или не запущен. Вернись к Шагу 1. |
| `Cannot connect to the Docker daemon` | Запусти Docker Desktop и подожди, пока кит перестанет мигать. |
| В тексте записей пишет «модели не найдены» | Не выполнен Шаг 5. Выполни обе команды скачивания моделей. |
| Долго «крутится» при первом запуске | Это нормально (скачивание). Просто подожди. |
| Хочу посмотреть, что происходит «под капотом» | `docker compose logs -f backend` (выход — `Ctrl + C`). |

## Адреса и порты (справка)

| Что | Адрес |
|-----|-------|
| Сайт CRM | http://localhost:3000 |
| Приём записей (для приложения-рекордера) | http://localhost:8080 |

Если CRM стоит на отдельном сервере (не на твоём компьютере), вместо
`localhost` сотрудники в рекордере указывают адрес/IP этого сервера, например
`http://192.168.1.50:8080`. Для серверного развёртывания см. файл **DEPLOY.md**.

## Что внутри (для технических специалистов)

Стек: Django REST Framework + React + PostgreSQL + Redis + Celery, локальная
транскрибация (faster-whisper + sherpa-onnx) и локальный анализ (Ollama).
Подробности развёртывания на сервере — в **DEPLOY.md**. Записывающее
приложение — отдельный проект **AudioRecorder**.
